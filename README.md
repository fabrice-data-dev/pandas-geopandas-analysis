# TP — Pandas & GeoPandas (UQTR)

Academic project (Data Science track) focused on **data cleaning with Pandas** and **geospatial analysis with GeoPandas**.

## What this project does

- Loads `data/villes.csv` (cities dataset)
- Applies data-quality rules (missing values, latitude/longitude bounds, population checks)
- Computes basic population statistics
- Exports a cleaned dataset to `outputs/`
- Builds a GeoDataFrame of cities (WGS84 / EPSG:4326)
- Loads a countries layer (`data/pays.*`) and runs a **spatial join**
- Exports join results (`CSV` + `GeoJSON`) to `outputs/`

## Repository structure

```
.
├─ data/                 # input datasets (CSV + country layer)
├─ outputs/              # sample outputs generated by the script
├─ src/                  # main application code
└─ tests/                # unit tests (pytest)
```

## Quick start

### Option A — Conda (recommended for GeoPandas)
```bash
conda env create -f environment.yml
conda activate tp2-geopandas
```

### Option B — pip
```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux: source .venv/bin/activate
pip install -r requirements.txt
```

## Run the application

```bash
python src/tp2_application_geo_g37.py
```

Outputs will be written to the `outputs/` folder.

## Run unit tests

```bash
pytest -q
```

## Notes (data layer)
GeoPandas usually expects a complete shapefile set (`.shp`, `.shx`, `.dbf`, and often `.prj`).
If your environment complains about missing sidecar files, replace `data/pays.*` with a complete layer (shapefile or GeoJSON)
while keeping the same filename (or adjust the path in the script).
